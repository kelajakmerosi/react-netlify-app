# Design Language

## Core Definition

The product's public-facing visual system is **Soft Academic SaaS**.

This is a calm, structured education-product aesthetic that feels professional, approachable, and easy to navigate. It should prioritize clarity, trust, and flow over visual novelty.

## Design Intent

- Build confidence quickly.
- Keep the next action obvious.
- Make learning, exams, and materials feel like one connected product.
- Use visual restraint instead of decorative overload.

## Visual Characteristics

- Light gray-blue backgrounds with white or near-white surfaces.
- One clear accent color, centered on the existing purple token family.
- Deep navy text for anchors and headings.
- Rounded geometry with soft corners and moderate depth.
- Subtle shadows, light borders, and gentle hover lift.
- Editorial or educational imagery with calm overlays.
- Strong hierarchy, quiet spacing, and predictable layout rhythm.

## Typography

- Prefer readable, friendly sans-serif type.
- Headings should feel confident and compact, not oversized or theatrical.
- Body copy should stay calm and utilitarian.
- Avoid overly condensed, futuristic, or luxury-styled type treatments.

## Layout Rules

- Design with clear sections and obvious grouping.
- Favor grid structure over freeform compositions.
- Keep primary actions visually dominant.
- Use secondary actions sparingly and with lower contrast.
- Mobile layouts should stack cleanly without changing the product's tone.

## Surface Rules

- Surfaces should feel soft and polished, not glossy or neon.
- Glass effects can be used as a component primitive, but the overall composition should not feel like a high-tech glassmorphism demo.
- Borders should be subtle and supportive.
- Shadows should suggest depth without becoming dramatic.

## Motion Rules

- Motion should be restrained and purposeful.
- Use small hover lifts, fades, and scale changes.
- Avoid flashy animation, large parallax effects, or aggressive blur-driven transitions.

## Color Behavior

- Use purple as the main highlight and interaction color.
- Support it with light neutral backgrounds and dark text.
- Keep strong contrast on headings and primary actions.
- Avoid introducing unrelated accent colors unless there is a clear semantic purpose.

## Product Matching Rule

- Landing, auth, and other first-impression pages must feel like they belong to the same product family.
- Auth pages should be a quieter extension of the landing page, not a separate futuristic or overly glassy interface.
- Internal app screens may use shared UI primitives like `GlassCard`, but composition should still respect the broader product tone.

## Do

- Use calm surfaces and clear hierarchy.
- Reuse the existing token system.
- Keep components readable and professional.
- Make interactions feel deliberate and low-friction.

## Avoid

- Neon gradients as the main personality.
- Dark-tech dashboard styling for marketing or auth screens.
- Overly glossy cards, heavy blur, or excessive floating effects.
- Competing accent colors or noisy backgrounds.
- UI that looks premium-first instead of learning-first.
